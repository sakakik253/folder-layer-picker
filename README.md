# Folder Layer Picker

深い階層構造を持つフォルダから、指定した階層のフォルダのみをルート直下に移動し、空になったフォルダを自動削除するPowerShell + WPFツール。

## 主な機能

- **階層分析機能**: フォルダ構造をスキャンして階層ごとの統計を表示
- **複数階層選択**: チェックボックスで複数の階層を同時に選択可能
- **柔軟な操作モード**: 4つのモードから選択
  - 移動＋空フォルダ全削除（デフォルト）
  - 移動のみ（階層構造を保持）
  - 削除のみ
  - カスタム（移動と削除を個別設定）
- **プレビュー機能**: 実行前に影響を階層別に予測表示
- **バックアップ機能**: 実行前に自動バックアップを作成
- **Undo機能**: バックアップから復元

## 使用方法

### 簡単起動（推奨）

**start.bat** をダブルクリックするだけで起動できます。

または、以下のバッチファイルを使用：
- `start.bat`: 通常起動
- `start-debug.bat`: デバッグモード（エラー詳細表示）
- `test-startup.bat`: 起動診断ツール

### コマンドラインから起動

```powershell
# PowerShell 7 (推奨)
pwsh -ExecutionPolicy Bypass -File .\folder-layer-picker.ps1

# Windows PowerShell
powershell -ExecutionPolicy Bypass -File .\folder-layer-picker.ps1
```

### 操作手順

1. **対象フォルダを選択**: 「参照」ボタンをクリックして対象フォルダを選択
2. **階層を分析**: 「分析」ボタンをクリックしてフォルダ構造をスキャン
3. **操作モードを選択**: 4つのモードから選択
4. **階層を選択**: 分析結果から階層をチェックボックスで選択（複数可）
5. **プレビュー確認**: 「プレビュー更新」ボタンで実行結果を確認
6. **実行**: 「実行」ボタンをクリックして処理を開始
7. **元に戻す**: 必要に応じて「元に戻す」ボタンでバックアップから復元

## 操作モード

### 1. 移動＋空フォルダ全削除（デフォルト）

選択した階層のフォルダをルート直下に移動し、すべての空フォルダを削除します。

**使用例**: 深い階層を一気にフラット化したい場合

### 2. 移動のみ

選択した階層のフォルダをルート直下に移動します。空フォルダは削除しません。

**使用例**: 階層構造を保持しながら特定の階層だけを移動したい場合

### 3. 削除のみ

選択した階層のフォルダを削除します。移動は行いません。

**使用例**: 不要な階層を削除したい場合

### 4. カスタム

移動と削除を個別に設定できます。削除範囲を細かく指定可能。

**使用例**: 複雑な整理作業をしたい場合

## 処理ルール

### 複数階層の選択

チェックボックスで複数の階層を同時に選択できます。

例: 階層1と階層2を同時に選択した場合、両方の階層のフォルダがルート直下に移動されます。

### フォルダの移動

指定した階層のフォルダをルート直下に移動します。

例: 階層2を選択した場合
```
元の構造:
  Root/
    ├─ 第1章/
    │   ├─ 1.1節/
    │   │   └─ 資料/
    │   └─ 1.2節/
    └─ 第2章/
        └─ 2.1節/

実行後:
  Root/
    ├─ 第1章_1.1節/
    ├─ 第1章_1.2節/
    └─ 第2章_2.1節/
```

### 同名フォルダの処理

同名のフォルダが存在する場合、元のパスをプレフィックスとして追加します。

- 形式: `{親フォルダパス}_{フォルダ名}`
- 例: `第1章\1.1節\資料` → `第1章_1.1節_資料`

### 空フォルダの削除

移動後、ファイルやサブフォルダが残っていない空のフォルダは自動的に削除されます（モードに応じて）。

## バックアップとUndo

### バックアップ

実行前に自動的にバックアップを作成します（オプションで無効化可能）。

- バックアップ先: `{元のフォルダ名}_backup_{タイムスタンプ}`
- 例: `MyFolder_backup_20251123_143022`

### Undo（元に戻す）

最新のバックアップから復元します。

1. 現在のフォルダ内容を削除
2. バックアップフォルダから復元
3. 元の状態に戻す

## ログファイル

処理の詳細はログファイルに記録されます。

- ファイル名: `folder-layer-picker_{日時}.log`
- 記録内容: 移動したフォルダ、削除したフォルダ、エラー情報など

- 管理者権限が必要な場合があります

## トラブルシューティング

### スクリプトの実行ポリシーエラー

```powershell
# 実行ポリシーを一時的に変更して起動
powershell -ExecutionPolicy Bypass -File .\folder-layer-picker.ps1
```

### XAMLファイルが見つからないエラー

- `MainWindow.xaml` と `folder-layer-picker.ps1` が同じフォルダにあることを確認してください

### 権限エラー

- 管理者権限でPowerShellを起動してください
- 対象フォルダへのアクセス権限を確認してください

### 文字化けが発生する場合

- スクリプトファイルはUTF-8で保存されています
- PowerShellのコンソール出力でのみ文字化けが発生する場合がありますが、UIとログファイルは正常に動作します

## 技術詳細

### アーキテクチャ

- **UI**: WPF (XAML) - Windows Presentation Foundation
- **ロジック**: PowerShell 5.1+
- **設計パターン**: イベント駆動型

### 主要関数

- `Get-FolderHierarchy`: フォルダ構造をスキャンして階層別に分類
- `Get-PreviewData`: 移動予測データを生成（名前衝突チェック含む）
- `Move-FoldersToRoot`: フォルダをルート直下に移動
- `Remove-EmptyFolders`: 空フォルダを再帰的に削除
- `New-Backup`: バックアップを作成
- `Restore-FromBackup`: バックアップから復元

## ユースケース

### 設計書の整理

設計書フォルダでは、階層のレベル感が重要です。柔軟な操作モードを使って最適な構造に整理できます。

**例: 章レベルは保持、節レベルを移動**

```
元の構造:
  設計書/
    ├─ 第1章_基本設計/
    │   ├─ 1.1_概要/
    │   ├─ 1.2_要件/
    │   └─ 1.3_アーキテクチャ/
    └─ 第2章_詳細設計/
        ├─ 2.1_データベース/
        └─ 2.2_API/

操作: 階層2を選択、「移動のみ」モード

実行後:
  設計書/
    ├─ 第1章_基本設計/     （保持）
    ├─ 第2章_詳細設計/     （保持）
    ├─ 第1章_基本設計_1.1_概要/
    ├─ 第1章_基本設計_1.2_要件/
    ├─ 第1章_基本設計_1.3_アーキテクチャ/
    ├─ 第2章_詳細設計_2.1_データベース/
    └─ 第2章_詳細設計_2.2_API/
```

### プロジェクトフォルダの整理

深すぎる階層を一気にフラット化します。

**例: 複数階層を同時に移動**

階層2、3、4をチェックして「移動＋空フォルダ全削除」で実行すると、すべてルート直下に移動され、使いやすいフラットな構造になります。

### 不要な中間階層の削除

特定の階層だけを削除して、階層を浅くします。

**例: 階層1だけを削除**

「削除のみ」モードで階層1を選択すると、階層1のフォルダが削除され、その配下（階層2）が自動的に階層1に昇格します。

## よくある質問 (FAQ)

### Q: 処理を実行した後、元に戻せますか？

A: はい。「バックアップを作成する」オプションを有効にしていれば、「元に戻す」ボタンで復元できます。

### Q: 複数の階層を同時に選択できますか？

A: はい。v2.0から複数階層の同時選択に対応しました。チェックボックスで複数選択してください。

### Q: ファイルも移動されますか？

A: いいえ。このツールはフォルダのみを移動します。ファイルは親フォルダと共に移動します。

### Q: 階層0とは何ですか？

A: ルートフォルダ直下のフォルダです。階層1は、階層0のフォルダ内のフォルダです。

### Q: バックアップはどこに作成されますか？

A: 対象フォルダと同じ親フォルダ内に、`{フォルダ名}_backup_{タイムスタンプ}` という名前で作成されます。
